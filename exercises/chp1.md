1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
     - 센서는 외부 환경을 감지해 전기 신호로 변환하는 입력 장치의 역할을 하고 액추에이터는 전기
       신호를 받아 물리적 동작을 수행하는 출력장치의 역할을 한다
     - 센서의 ex) 온도센서, 압력센서                       액추에이터의 ex) 모터, LED, 스피커   
  
  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
     - 가장 큰 이유는 역할과 환경이 다르기 때문이고 디바이스의 경우 메모리, CPU가 매우 제한되고
       실시간 처리가 중요하고 하드웨어를 직접 제어해야 하므로 C, C++ 같은 Low-Level Language를
       사용하고 대시보드의 경우 자원이 풍부하고 사용자 인터페이스, 데이터 분석, 네트워크 처리
       중심이기 때문에 Python, JavaScript와 같은 High-Level Language를 사용한다.
  
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
     - 한줄 요약을 하면 유연하고 확장성 높은 시스템을 만들 수 있다. 정도이고 밑에 5가지 이점이 있다.
       1. 결합도 감소: 디바이스와 대시보드가 서로 직접 연결 하지 않아도 되어 서로 구조가 바뀌어도
          영향이 적다.
       2. 확장성 향상: 디바이스 수가 늘어나도 브로커가 중간에서 분산처리를 실행해 대규모 시스템에
          유리하다.
       3. 비동기 통신 지원: 송신자 또는 수신자가 동시에 접속 안해도 메세지를 전달 할 수 있어 안정적임
       4. 부하 분산 및 성능 향상: 브로커가 메시지 큐잉, 라우팅 처리로 시스템의 전체 효율이 증가한다.
       5. 유지보수 용이: 디바이스나 대시보드에서 한 쪽이 수정을 해도 전체 시스템에는 영향을 적게
          받는다.
